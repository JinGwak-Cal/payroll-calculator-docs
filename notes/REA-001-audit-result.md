## D4. Patch Specification (AE=N 항목만)

### Patch-01
- Target: A-04 답변 정직성 규칙
- 문제: "예측 금지/정정 안 하는 것 금지"에 구체 Action 없음
- Root Cause: 선언형으로만 작성됨
- Before: "확실하지 않으면 즉시 '불확실'을 먼저 선언한다."
- After 제안: "답변 시작 문장에 확신도(확실/추정/불확실) 표기를
  먼저 쓴다." (R19 Assertion Gate와 통합 가능)
- 적용 위치: 구조4-목차01
- Expected Effect: 정직성 규칙이 R19와 사실상 같은 메커니즘으로 통합됨
- Regression Risk: 낮음(R19와 중복 위험만 있음 — 신설 대신 R19에 흡수 권장)

### Patch-02
- Target: A-08 §운영-1 자동 판단 금지
- 문제: "명시적으로 요청된"의 판정 기준 없음
- After 제안: §운영-4(생성 트리거 단어 목록)를 §운영-1의 판정
  기준으로 명시적으로 연결하는 문장 추가
- Regression Risk: 낮음(이미 있는 §운영-4를 참조만 시키면 됨 — 신규 규칙 아님)

### Patch-03
- Target: R3/R5 (토큰절약 원칙 내)
- 문제: "결론 먼저", "불확실하면 확인" — 형식 기준 없음
- After 제안: 협업 응답 원칙(A-20, "①결론②조건③근거④세부")을
  R3의 실행 형식으로 명시 연결. R5는 "확실/추정/불확실" 3단
  표기 도입(R19와 공유)
- Regression Risk: 낮음

### Patch-04

- 문제: "적용됐다고 간주하지 않는다", "최소만", "완전한 문장",
  "요약으로 대체 금지" — 전부 판정 기준 없는 선언형
