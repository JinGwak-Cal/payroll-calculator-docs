# UI Audit 기준 v1

목표 이미지: 새롭고 신선하고 깔끔하고 잘 정돈된 이미지
상태: 확정 (디자인 방향 검토 단계)

## Audit 8항목

| # | 항목 | 평가 질문 |
|---|------|-----------|
| 1 | 가독성 | 한눈에 이해되는가 |
| 2 | 일관성 | 다른 화면과 같은 규칙인가 |
| 3 | 단순성 | 불필요한 요소가 없는가 |
| 4 | 신뢰성 | 계산 결과가 중심인가 |
| 5 | 현대성 | 2026년 앱처럼 보이는가 |
| 6 | 모바일 우선성 | 작은 화면에서도 편하게 사용할 수 있는가 |
| 7 | 접근성 | 중년 사용자도 입력값과 결과값을 쉽게 읽을 수 있는가 |
| 8 | 정보 밀도 | 한 화면에 정보가 너무 많지는 않은가 |

## 크기 기준

크게 보여야 할 것:
- 계산 결과 금액
- 입력 숫자

작아도 되는 것:
- 설명문
- 보조정보
- 참고문구

## 적용 순서

UI Audit 1차 전수조사
↓
기준점 선정
↓
앱 전체 공통 UI 기준 확정
↓
기능별 순차 적용 (주휴 → 연차 → 나머지 수당 → 전체 UI)
