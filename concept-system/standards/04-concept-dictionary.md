# 04. Concept Dictionary v1.0

> Type 분류 체계의 정의. 카드 작성 시 "이건 Rule인가?"가 아니라 "Rule의 정의에 부합하는가?"로 판단한다.

---

| Type | 정의 (이 개념은 무엇에 답하는가) | 코드 |
|---|---|---|
| **Theory** | 왜(Why)를 설명하는 개념 | T |
| **Hypothesis** | 아직 검증되지 않은 가설 — 반복·재현 검증이 필요한 주장 | H |
| **Case Study** | 실제 관찰된 단일 사례 기록. 일반화하지 않은 원시 관찰 | CS |
| **Rule** | 무엇을 해야 하는가를 규정하는 개념 — 권한·제약·판단 기준을 다룬다 | R |
| **Workflow** | 어떤 순서로 진행되는가를 설명하는 개념 — 상태 전이와 절차를 다룬다 | WF |
| **Metric** | 얼마나를 측정하는 개념 — 무엇을, 어떻게 측정하는가를 다룬다 | M |
| **Architecture** | 어떻게 구성되는가를 설명하는 개념 | A |
| **Business** | 어떻게 가치와 수익을 만드는가를 설명하는 개념 | B |

기존 "Research" 단일 분류는 **Hypothesis / Case Study로 분리**한다. 판단 기준: 가설·이론 후보면 Hypothesis, 1회성 관찰·사례면 Case Study.

**Policy(운영 정책) 분류는 신설 보류, 후보로만 남긴다.** 자세한 사항은 [02. Concept Manual] 참조.
