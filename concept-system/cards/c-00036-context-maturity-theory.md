# C-00036 : 운영 Context 성숙 이론 (Context Maturity Theory)

| 그룹 | 필드 | 내용 |
|---|---|---|
| **식별** | Concept ID | C-00036 |
| | Coordinate | T-02 |
| **기본** | 이름(한글) | 운영 Context 성숙 이론 |
| | 영문명 / 약어 | Context Maturity Theory |
| **정의** | 공식 정의 | 프로젝트의 운영 Context 자산(SoT 문서·decisions·current-step·Review History 등)이 충분히 누적·정리된 상태가 AI 자율성 발생의 전제조건이라는 이론 |
| | 목적 | Context Threshold(C-00016) 가설이 발생하는 구체적 조건(무엇이 갖춰져야 임계점에 도달하는지)을 실무적으로 정의한다 |
| | 핵심 질문 | AI 자율성이 발생하려면 어떤 운영 조건이 먼저 갖춰져야 하는가 |
| **분류** | Type | Theory *(운영복리효과(C-00035)가 "왜 비선형으로 향상되는가"를 설명하는 원리라면, 본 이론은 "어떤 조건이 갖춰져야 향상이 시작되는가"를 설명하는 전제조건 이론 — Hypothesis가 아닌 Theory인 이유: 본 프로젝트 전체 운영에서 SoT 관리와 자율성 향상의 상관관계가 반복 확인됨)* |
| | Status | Candidate |
| | Stability | Emerging |
| **관계** | 상위 개념 | 없음 *(C-00035와 동급의 설명 이론 — 향후 더 상위 Theory가 생기면 재배치)* |
| | 하위 개념 | 없음 |
| | 관련 개념 | C-00035 (운영복리효과 — 성숙이 시작된 후 발생하는 비선형 효과), C-00016 (컨텍스트 쓰레시홀드 — 성숙의 결과), C-00012 (Context Layer — 성숙이 일어나는 공간) |
| | Before(전 단계) | 없음 |
| | After(후 단계) | C-00035 (운영복리효과 — 성숙 조건 충족 후 복리 효과 시작) |
| **비교** | 기존 유사 개념 | CMMI (Capability Maturity Model Integration), Organizational Learning Theory |
| | 차이점 | CMMI는 조직 프로세스 성숙도를 다루지만, 본 이론은 "AI 협업을 위한 운영 문서(SoT·decisions·current-step)의 성숙도"라는 훨씬 좁고 구체적인 맥락에 한정된다 |
| | 용어 유형 | 기존용어조합+신개념 |
