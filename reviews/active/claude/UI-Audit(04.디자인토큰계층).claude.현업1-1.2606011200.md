# UI-Audit (04. 디자인토큰계층)

**목적**
어떤 UI든 레벨 1~4에서 속하는 곳만 결정하면 디자인이 자동 결정된다.
개별 디자인 결정 불필요.

---

## 레벨 1 — 기본 토큰 (전체 공통)

앱 전체에 공통 적용되는 가장 기본적인 토큰.

- 앱 배경색
- 카드 배경색
- 의미색 (실수령=emerald / 공제=red)
- 중립색 (텍스트/테두리/배경 계층)

※ 상태(선택/활성/비활성)는 의미색 사용 금지. 레벨 3에서 폰트·테두리·명암으로 처리.

## 레벨 2 — 컴포넌트 토큰 (컴포넌트 공통)

동일 종류의 UI 컴포넌트에 공통 적용.

| 컴포넌트 | 적용 토큰 |
|----------|-----------|
| 카드(Card) | radius / border / padding / shadow |
| 버튼(Button) | radius / padding / font |
| 칩(Chip) | radius / border / padding / font |
| 입력창(Input) | radius / border / padding / font |
| 탭(Tab) | radius / padding / font |
| 토글(Toggle) | 크기 / radius |

## 레벨 3 — 상태 토큰 (상태 공통)

§원칙-7 실현. 색 없이 폰트·테두리·명암으로만 표현.
앱 전체 모든 상태 표현에 동일 적용.

| 상태 | 표현 방식 |
|------|-----------|
| 선택/활성 | font-semibold + 진한 텍스트 + 진한 테두리 |
| 미선택/비활성 | font-normal + 회색 텍스트 + 연한 테두리 |
| 성공 | emerald (의미색) |
| 경고 | amber |
| 오류/공제 | red (의미색) |

## 레벨 4 — 레이어 토큰 (화면 레이어별)

화면의 레이어(단계)별 위계 기준.

| 레이어 | 타이포 위계 | 여백 기준 |
|--------|-------------|-----------|
| 결과 레이어 | 결과금액 최대 | 넓게 |
| 입력 레이어 | 입력값 중간 | 표준 |
| 상세 레이어 | 보조정보 최소 | 좁게 |

---

## 사용 원칙

1. UI 결정 시 레벨 1→2→3→4 순서로 확인
2. 해당 레벨 토큰 자동 적용
3. 개별 색상/폰트/여백 직접 지정 금지

---

## 미확정 항목 (실제 값)

다음 항목의 구체적 값은 UI-Audit(05.토큰값확정)에서 결정:
- 배경색 hex/class
- 카드 radius 값 (rounded-2xl vs rounded-3xl)
- 타이포 스케일 구체값
- 의미색 구체 shade
