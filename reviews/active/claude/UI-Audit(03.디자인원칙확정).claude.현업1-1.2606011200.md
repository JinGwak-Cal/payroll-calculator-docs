# UI-Audit (03. 디자인원칙확정)

**최상위 목표**
> 새롭고 신선하고 깔끔하고 잘 정돈된 이미지

---

## §원칙-0 영역별 일관화·통일화 (최상위 원칙)

모든 하위 원칙은 이 원칙을 실현하기 위한 수단이다.
일관화·통일화의 단위는 UI의 **4축 주소**(Type·Component·Feature·Layer)이며,
같은 좌표를 공유하는 UI는 동일 규칙을 적용한다.
상세 값 목록은 **UI-Audit(05)를 정본으로 참조**한다. (03에는 값 목록 전체를 중복 기재하지 않음)

- **Type(존재 목적)**: 같은 목적의 UI는 같은 상호작용·표현 규칙 적용
- **Component(전체 공통)**: 같은 종류의 UI는 앱 전체에서 동일 규칙 적용
- **Feature(기능별)**: 주휴·연차·야간·연장 등 같은 역할의 UI는 같은 디자인 언어 사용
- **Layer(레이어별)**: 입력→결과→상세 각 레이어는 동일 위계·구조·강조 방식 유지

## §원칙-1 결과가 항상 중심

결과값 > 입력값 > 보조정보 위계를 모든 화면에서 유지한다.
결과 금액이 화면에서 시각적으로 가장 지배적이어야 한다.

## §원칙-2 한 의미 = 한 색

실수령·가산·공제 등 각 의미에 색상 1:1 고정.
같은 색이 다른 의미로 쓰이는 것 금지.

## §원칙-3 UI 컴포넌트는 같은 언어

Card·Button·Toggle·Input·Tab·Badge·Chip 등
동일 종류의 UI는 동일 토큰(radius·border·padding·shadow·spacing)을 사용한다.

## §원칙-4 설명은 필요한 순간에만

설명은 상시 노출보다 필요한 순간에만 노출한다.
결과값·입력값이 화면을 지배하고, 안내문구는 최소화한다.

## §원칙-5 중년 친화

결과값·입력값은 크게. 보조설명은 작게.
작은 화면(모바일)에서도 핵심 숫자는 읽혀야 한다.

## §원칙-6 가로 우선

가능하면 한 줄에 표현한다.
세로 확장은 이해도 향상에 필요한 경우에만 허용한다.

## §원칙-7 상태는 같은 언어

선택/미선택/활성/비활성/확정/미확정은 앱 전체에서 동일한 디자인 규칙을 사용한다.
- 선택/활성: font-semibold + 진한 텍스트 + 진한 테두리
- 미선택/비활성: font-normal + 회색 텍스트 + 연한 테두리
- 색상 사용 금지. 폰트·테두리·명암으로만 표현한다.

적용 범위: 수당 선택 칩 / 탭 / 토글 / 필터 / 즐겨찾기 / 주휴 선택 등 앱 전체
