## 판단 근거 및 과정 요약

### 왜 이 작업을 시작했나
S-1 주휴 체크박스 버그 수정 완료 후, 병합주/이전기간 판단 모델이
사용자 혼란을 유발한다는 판단 하에 설계 변경 착수.
단순 버그 수정이 아닌 주휴 시스템 설계 철학 변경에 해당.

### STEP 1-A 전수조사에서 발견한 것
키워드 검색으로 시작했으나 전수조사 과정에서
키워드 목록에 없던 신규 변수 발견:
- includeFirstWeek / includeLastWeek — calcJuhuByWeek에 직접 전달
- isEdge — 계산 함수 내부에서 직접 생성, weekDetails로 전달
- PrevWeekState / prevWeekOn — 이전기간 토글 전담 구조
- week.isFullWeek — 계산 분기에 사용

### STEP 1-B 의존도 분류 핵심 판단
calcJuhuByWeek 함수 내부 확인 결과:
- isMergedFirst는 함수 내부에 없음 (호출 전 분기에 존재)
- 계산 제어는 includeFirstWeek / includeLastWeek 두 파라미터로만 작동
- 따라서 이 두 값의 기본값만 고정하면 계산 엔진 무변경 가능

### 왜 A안(완전 제거)을 보류했나
- calcJuhuByWeek 시그니처 수정 필요
- 상위 호출부 수정 필요
- 결과 데이터 구조 수정 필요
- 범위가 크고 위험도 높음

### 왜 B안(고급옵션 이동)을 2순위로 뒀나
- 계산/UI 구조 유지 가능
- 하지만 복잡성과 사용자 혼란이 유지됨
- 법리 세부 제어 보존이 필요할 때 재검토

### 왜 C안(UI 비노출)을 1순위로 확정했나
- includeFirstWeek / includeLastWeek 기본값 true 고정만으로 충분
- calcJuhuByWeek 함수 건드리지 않음
- 작업량 최소, 리스크 최소, 롤백 쉬움
- 사용자 혼란 제거 목표 달성 가능

## 안내문구 확정안

### 공통 상단 안내 (고정)
각 주차의 주휴수당은 개근/결근 선택과 주 15시간 이상 여부에 따라 계산됩니다.
첫 주와 마지막 주도 주휴수당 대상 여부에 따라 직접 선택해 주세요.

### 주차별 안내 — 4가지만 유지
| 상태 | 표시 문구 |
|---|---|
| 주휴대상 주차 없음 | 주휴수당 대상 주차가 없습니다. |
| 개근 + 15시간 이상 | 주휴수당이 반영됩니다. |
| 개근 + 15시간 미만 | 주 15시간 미만으로 주휴수당에서 제외됩니다. |
| 결근 선택 | 결근으로 선택되어 주휴수당에서 제외됩니다. |
