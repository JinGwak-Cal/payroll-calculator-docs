# Premium 입력 구조 재설계 (01. 구조확정)

---

## §구조-1 계산 진실원천

시스템 A(use-calc / calc-engine)가 유일한 계산 원천이다.
grossPay / netPay / 공제 / 주휴 / 저장 / 불러오기 모두 A에 연결된다.
시스템 B의 독자 계산(totalPremium)은 제거한다.

## §구조-2 입력기 역할 분리

시스템 B(PremiumSection)는 입력 UI 전용으로 전환한다.
[연장][야간][휴일][맞춤] 조합 선택 UX는 재사용한다.
선택 결과는 A state(includeNight / includeOvertime / includeHoliday + hours + rate)로 전달된다.

## §구조-3 조합 자동 해석

사용자는 [연장][야간][휴일][맞춤] 버튼만 선택한다.
- 1개 선택 → 단일가산 자동 해석
- 2개 선택 → 이중가산 자동 해석
- 3개 선택 → 삼중가산 자동 해석
- 맞춤 선택 → 가산율 직접 입력

사용자는 "단일/이중/삼중"이라는 용어를 몰라도 된다.

## §구조-3-1 입력 단위 정의

근무내역 1건 = 근무상황 조합 + 근무시간
같은 조합도 시간이 다르면 별개 내역이다.

예:
```
① 연장+야간 · 2시간
② 연장+야간 · 3시간
③ 휴일 · 4시간
```

## §구조-4 수당 선택 UI — 칩 토글

체크마크 없음. §원칙-7에 따라 폰트·테두리·명암으로만 표현한다.

```
선택:   [ 연장 ]  font-semibold + 진한 텍스트 + 진한 테두리
미선택: [ 야간 ]  font-normal + 회색 텍스트 + 연한 테두리
```

칩을 다시 누르면 즉시 해제된다.
칩 상태 자체가 요약이다. 별도 요약 텍스트 불필요.
시간 입력 한 줄: 근무시간 [ 3 ] 시간

## §구조-5 역할 분리 및 내역 구조

- 칩 = 근무상황 선택/해제
- \+ = 새 근무내역 추가

여러 내역은 칩 상태 자체가 요약 (각 내역별 선택 상태가 다름):
```
①
[연장✓] [야간✓] [휴일] [맞춤]  시간 [3]

②
[연장] [야간] [휴일✓] [맞춤]   시간 [4]

[ + 근무내역 추가 ]
```

## §구조-5-1 내역 수정 UX

기존 내역은 항상 편집 가능한 상태로 유지한다.
- 칩 터치 시 즉시 선택/해제된다.
- 시간 입력 변경 시 즉시 반영된다.
- 별도 편집모드 진입 없음
- 별도 Drawer 재오픈 없음
- 별도 저장 버튼 없음
- 모든 변경은 실시간 반영된다.

## §구조-6 ResultGrid 역할

ResultGrid는 결과 표시판이자 입력 진입점이다.
기존 인라인 펼침 입력 UI → 제거
체크박스 역할 → 재검토 예정 (입력 연동으로 자동화 가능)

## §구조-6-1 진입 가능 행

가산수당 영역 터치 → 입력기 열림
실수령액 / 총급여 / 기본급 / 주휴수당 / 연차수당 / 공제 → 진입 불가
세부 행 구분은 실태조사 후 최종 확정

## §구조-7 화면 구조

- 상단 영역: ResultGrid (실수령 / 총급여 / 수당 항목 / 금액)
- 하단 영역: 입력 Drawer (칩 + 시간 입력 + 내역 목록)
- 입력 즉시 → ResultGrid 실시간 반영

## §구조-8 디자인 일관화

ResultGrid와 입력 Drawer는 하나의 기능처럼 보여야 한다.
§원칙-0 영역별 일관화 + §원칙-6 가로 우선 + §원칙-7 상태는 같은 언어
첫 번째 실제 적용 지점.

## §구조-9 저장/공유

- 저장: 시스템 A state 기반 유지 (기존 호환)
- 공유: 현재 가산 정보 미포함 → **구현 전 영향도 검토 필수**
