## 3. 시간속성 수당 — Drawer 입력 구조 확정

### 3-1. ResultGrid Row 구조 (상단 고정)

```
┌──────────────────────────────────────┐
│  야간수당        +32,000원           │
│  ←── Row 전체 탭 = Drawer 열림 ────→ │
└──────────────────────────────────────┘
```

- Row 전체 탭 → 하단 Drawer 열림
- 시간값 없음 = 계산금액 0 (자동 OFF)
- 시간값 입력 = 계산 반영 (자동 ON)
- 우측 토글: 일시 제외 기능 후보 — 보류
- **체크박스(☑/☐) 제거 후보** — 대체 ON/OFF 방식은 STEP6+ 보류
- RESULT-04: 체크박스 방식 제거 확정, 대체 ON/OFF 방식 STEP6+ 보류

### 3-2. Drawer 위치 및 동작

- **항상 하단 고정** (Bottom Sheet)
- ResultGrid(상단)는 Drawer가 열려도 계속 보임
- Drawer 닫기: 배경 탭 또는 아래로 스와이프

### 3-3. Drawer 내부 흐름 (수정 1 반영 — 칩 사라지는 시점 명시)

```
STEP 1 — 어떤 수당이에요?

  선택 중 상태 (완료 전):
  [연장✓] [야간] [휴일] [완료]     ← 비선택 칩 유지
                                      여러 개 선택 가능

  완료 버튼 누른 후:
  [연장✓] [완료]                   ← 비선택 칩 제거
                                      선택된 칩만 표시

  → [완료] 는 [연장][야간][휴일] 과 같은 줄에 배치
  → 선택된 칩 다시 탭 = 선택 해제 (완료 전까지 가능)

STEP 2 — 몇 시간이에요?             ← 시간 UI 먼저
  [ ◀◀ ][ ◀ ][ 4시간 ][ ▶ ][ ▶▶ ]
  숫자 탭 → 키패드 직접입력 전환 (토스 방식)

STEP 3 — 가산율
  "기본시급의 50% 추가됩니다"         ← 표준 안내문구 (선택개수 자동결정)
  [맞춤 가산율로 변경 →]             ← 문구 옆, 작게

즉시 결과:
  "+32,000원이 야간수당으로 추가됩니다"

[이 수당 추가하기]
```

### 3-4. 요약 행 편집 (수정 2 반영 — 모든 값 수정 가능)

```
[행 요약] 야간 4h · 50%추가 · +32,000원
  → 탭하면: 편집모드 복귀 (Drawer가 해당 행 데이터 로드)
  → 기존 값(시간 / 가산율 / 수당종류) 모두 유지
  → 시간, 가산율, 수당종류 모두 수정 가능
  → 초기화는 별도 삭제/초기화 동작으로 분리
```

- Drawer는 항상 하단 고정 — 수정 대상 행 위치와 무관

### 3-5. 가산율 분기

| 구분 | 가산율 | UI |
|---|---|---|
| 표준 단일 (1개) | 50% 고정 | 안내문구만 |
| 표준 이중 (2개) | 100% 고정 | 안내문구만 |
| 표준 삼중 (3개) | 150% 고정 | 안내문구만 |
| 맞춤 | 직접입력 | [맞춤 가산율로 변경 →] 탭 후 노출 |

- 가산율 표기: `50%추가/100%추가/150%추가` 통일 (manual-v14 §G)
- 표준/맞춤 판정: `premiumRate === selectedAllowances.length * 50`

### 3-6. 여러 수당 추가

```
[행 1 요약] 야간 4h · 50%추가 · +32,000원
[행 2 요약] 연장+야간 2h · 100%추가 · +24,000원
──────────────────────────────────────────────
[다른 수당 추가]
──────────────────────────────────────────────
[현재 입력 중 Drawer]                            ← 항상 하단 고정
  [연장]  [야간]  [휴일]  [완료]
```

- 추가 버튼 문구: **[다른 수당 추가]** (확정)
- **최대 행 수: 보류** (수정 3 반영 — 연장/연장+야간/연장+야간+휴일/맞춤 조합만 해도 4행 필요)

### 3-7. 동일수당 중복 규칙

- **허용 — 소프트 경고**: 시간구간이 다른 경우 동일 수당 여러 행 허용
  - 예: `[연장] 4h 50%` + `[연장,야간] 3h 100%` — 정상 (18:00~22:00 / 22:00~01:00)
  - UI: 확인 다이얼로그 후 허용
- **금지 — 하드 차단**: 동일 수당의 표준↔맞춤 동시 적용
  - MVP 시간구간 필드 부재 → 시간구간 구분 불가 → 기본 하드 차단
  - 시간구간 필드 도입 시 동일 시간구간 기준으로 재검토

### 3-8. 행의 의미 — 설계 철학 확정

> **행 = 근무상황 1건** (수당 총합 입력기가 아님)

이 철학 하에:
- 동일 수당이 여러 행에 등장 가능 (시간구간이 다른 경우)
- 미래 자동 시간대 계산(startTime/endTime)과 호환
- 근무지합산 시 selectedAllowances 기준 수당 계열 필터링 가능

## 7. 저장값 구조

```json
{
  "allowanceRows": [
    {
      "id": "row-1",
      "selectedAllowances": ["야간"],
      "premiumRate": 50,
      "premiumHours": 4
    },
    {
      "id": "row-2",
      "selectedAllowances": ["연장", "야간"],
      "premiumRate": 100,
      "premiumHours": 3
    }
  ]
}
```

저장하지 않는 값: `premiumAmount` / `premiumType` / `expectedRate` / `isStandard` / `isCustom` / `allowanceCombo` / `mode`

## 8. 확정/보류 종합

### 확정
- 영역 3분리 (상태유급형→시간속성→공제차감)

- 칩 처리: 완료 전 비선택 유지, 완료 후 비선택 제거
- 재탭 편집: 모든 값(시간/가산율/수당종류) 유지, 전부 수정 가능
- 추가 버튼: [다른 수당 추가]
- 행 철학: 행 = 근무상황 1건
