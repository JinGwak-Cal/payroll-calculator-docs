## 2. 전체 화면 종합 순위 (정성·가중 기반)

1. **NightPayCard** — 위계·여백·rounded-3xl·그리드 정렬 모범. 목표 이미지 최부합.
2. **PaySimulator** — 현대성·임팩트 최고. 단, 메인 결과 위계 역전·팔레트 밖 색 감점.
3. **저장/기록** — 큰 결함 없는 안정적 일관성.
4. **Home 입력** — 입력 완성도 높음. PerDayInput·StickyHeader 토큰 이탈이 발목.
5. **연장 Premium** — 카드군 모듈 일관성 모범. 활성색 이탈·작은 가산금액 감점.
6. **주휴 Juhu** — JuhuPanel 우수 ↔ JuhuSection 약함. 순효과 중위.
7. **연차 Annual** — 5색 노이즈로 일관성 최저급 + 작은 결과금액.
8. **휴일 Holiday** — 전용 UI 부재. 인지·위계 최약.
9. **ResultGrid** — 최하위. 중심 화면이 중심 기능 실패 + 과밀 + 토큰 분산.

## 3. 기준점(baseline) 후보 3개 (최종 선정 보류)

**A. NightPayCard**
- 강점: 위계·여백·rounded-3xl·그리드 정렬 모범. 큰 값.
- 약점: slate↔gray 색 이탈. card-in-card 구조 밀도 소지. 색-상태 의미 표현 미검증.

**B. PaySimulator**
- 강점: 현대성·임팩트. 목표 "신선함"에 최적. 최대 금액 표현.
- 약점: gradient·차액색(green/rose)이 표준 팔레트 밖. 메인 결과 위계 제약 전제.

**C. WageInput / ModeToggle**
- 강점: focus-within 전환. 큰 입력값. 명확한 위계.
- 약점: 입력부에 한정(결과 카드 미포괄). 단위 텍스트 작음.

*(참고 4순위) JuhuPanel 금액 카드 — emerald·큰 값·rounded-3xl로 결과금액 카드 표준 후보.*

## 4. 우선 정리 대상 최종 3개

1. **ResultGrid** — 영향도 1위. 헤드라인 금액=섹션 라벨 동일 크기. 과밀(py-1.5). radius·색 혼재. 중심 화면이라 수정 체감 최대.
2. **AnnualLeaveSection** — 일관성 최저. 단일 컴포넌트 5색 + 결과 text-sm.
3. **JuhuSection 탭** — 도메인 내 색 언어 분리 + 다주차 탭 축소. 통일 레버리지 큼.
   *(4순위: PerDayInput — 입력부 국소 토큰 이탈)*

---

## 5. 통일 시 선확정 디자인 토큰 3가지

1. **타이포 스케일(위계 토큰)** — 1순위.
   최종 결과 금액 > 상세 금액 > 보조정보 순 위계 정상화.
   이유: 현재 역전(16px < 25.6px < 30px)이 ResultGrid 결함의 근본 원인.

2. **색 의미 사전(semantic color)**
   활성/실수령/가산/공제/경고를 1:1 고정.
   이유: 분산 확인(indigo 15·blue 7·emerald 3·amber 6·red 10 파일). 연차·주휴·연장 일관성 결함 동시 해소.

3. **카드 Radius 표준**
   최상위 카드 단일 radius 확정.
   이유: 3xl ↔ 2xl 혼재 확인. "통일감" 가장 가시적. 표준화 비용 낮음.
   *(구체 값은 기준점 결정과 연동, 보류)*
